# Experiments

Each subdirectory is one experiment (a named parameter sweep comparing
Nominal, UpperBound, and BudgetRobust station-selection models).

## Directory layout

```
experiments/
└── <YYYY-MM-DD_experiment_name>/
    ├── base.toml            # fixed config shared by all jobs
    ├── sweep.toml           # swept parameters (Cartesian product)
    ├── config/              # generated by 01_setup.jl
    │   ├── job_list.csv
    │   ├── jobs.txt
    │   └── job_<N>.toml
    ├── runs/                # one subdirectory per completed job
    │   └── <timestamp>_<model>_<hash>/
    │       ├── config.toml
    │       ├── metrics.json
    │       ├── logs.txt
    │       └── <exported solution files>
    ├── analysis/            # output of aggregate.jl
    │   └── results.csv
    ├── setup_metadata.json
    └── 02_submit.sh         # copy of scripts/02_submit.sh, edited for this exp
```

## Workflow

```bash
# 1. Create experiment directory and write base.toml + sweep.toml
mkdir experiments/2026-05-01_gamma_sweep
# ... write base.toml and sweep.toml ...

# 2. Generate per-job configs
julia scripts/01_setup.jl experiments/2026-05-01_gamma_sweep

# 3a. Run locally (single job, for testing)
julia scripts/03_run_job.jl experiments/2026-05-01_gamma_sweep 1

# 3b. Or submit to SLURM
cp scripts/02_submit.sh experiments/2026-05-01_gamma_sweep/
# edit EXPERIMENT and --array in the copy, then:
sbatch experiments/2026-05-01_gamma_sweep/02_submit.sh

# 4. Aggregate results
julia scripts/aggregate.jl experiments/2026-05-01_gamma_sweep
```

## Model types

| `model.type`       | Description |
|--------------------|-------------|
| `NominalModel`     | Uses E[q_ods] (scenario-average demand). Baseline. |
| `UpperBoundModel`  | Uses q̄_ods for every OD pair. Maximally conservative. |
| `BudgetRobustModel`| Budget uncertainty set with Gamma_s per scenario type. |

## Key parameters

| Parameter | Location | Description |
|-----------|----------|-------------|
| `k` | `[parameters]` | Active stations per scenario |
| `l` | `[parameters]` | Total stations to build (≥ k) |
| `lambda` | `[parameters]` | In-vehicle travel time weight λ |
| `gamma` | `[parameters]` | Budget Γ_s (scalar or per scenario type) |
| `demand_bounds_file` | `[data]` | CSV with q_low/q_high per OD pair and scenario type |

## Out-of-sample evaluation

After optimizing, `03_run_job.jl` calls `evaluate_out_of_sample()` (to be
implemented) which fixes the solution (y, z, x) and computes realized cost
on held-out demand samples.  `aggregate.jl` then reports:

- mean realized cost
- 90th / 95th percentile realized cost
- worst-case realized cost
- price of robustness = (robust objective − nominal objective) / nominal objective
