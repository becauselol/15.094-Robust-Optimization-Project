\begin{abstract}
We extend a two-stage stochastic virtual bus stop (VBS) selection model to
handle uncertain passenger demand within each scenario type (morning,
afternoon, evening, night).  Demand for each origin--destination pair
$(o,d)$ and scenario type $s$ is modeled as lying in a capped-total-demand
uncertainty set parameterized by per-OD upper bounds together with a
per-scenario upper bound on aggregate demand; lower bounds are fixed at zero
by design.  We derive the dual robust counterpart of the inner maximization,
which remains linear and can be solved with the same MILP machinery as the
nominal model.  We compare the nominal base model with its robust
total-demand-cap counterpart on out-of-sample demand realizations and report
average cost, high-percentile (P90/P95) cost, and the price of robustness.
\end{abstract}

\section{Introduction}
\label{sec:intro}

Microtransit systems offer flexible, on-demand shared rides between a set of
designated virtual bus stops (VBS).  A key design decision is which candidate
locations to build as VBS.  Prior work formulates this as a two-stage stochastic
mixed-integer program: in the first stage, build $l$ stations and choose which
$k \le l$ are active in each scenario $s \in S$; in the second stage, assign
passengers to origin--destination VBS pairs.

A limitation of the stochastic model is that it treats the demand $q_{ods}$
within each scenario as a fixed, known quantity (typically the historical mean).
In practice, demand fluctuates even within a scenario type: morning-peak demand
on one day may differ substantially from the next.  This motivates a
\emph{robust} extension where $q_{ods}$ is uncertain within a bounded set.

The uncertainty model adopted here is designed to encode a direct operational
statement: each OD pair has a plausible demand range, and the \emph{total}
demand within a scenario type should not exceed a calibrated upper bound.  This
produces a tractable robust counterpart for the base station-selection model.

Accordingly, the report focuses on two formulations only:
\begin{itemize}
  \item the nominal base model, which optimizes against scenario-average demand;
  \item the robust counterpart, which protects against worst-case demand
        realizations within a capped uncertainty set.
\end{itemize}

Under a nominal-demand optimization, the chosen station set may fit the average
demand pattern well but perform poorly when demand shifts toward OD pairs that
are less conveniently located relative to the selected stops.  The robust model
reduces this sensitivity by favoring station configurations whose assignment
performance remains more stable across plausible demand realizations.

\section{Base Model}
\label{sec:base}

\subsection{Decision variables}

\begin{align*}
  y_j           &\in \{0,1\} & &\text{build station } j \\
  z_{js}        &\in \{0,1\} & &\text{station } j \text{ is active in scenario } s \\
  x_{odjks}     &\in \{0,1\} & &\text{assign OD pair } (o,d) \text{ to origin VBS } j,
                               \text{ destination VBS } k \text{ in scenario } s
\end{align*}

\subsection{Nominal objective}

The nominal two-stage objective minimizes expected passenger travel cost:
\begin{equation}
  \min_{x,y,z}
  \sum_{s \in S} \sum_{(o,d) \in \Omega} \sum_{j,k \in J}
  q_{ods}\,
  \underbrace{\bigl(d^{\mathrm{orig}}_{oj} + d^{\mathrm{dest}}_{dk} + \lambda\,c_{jk}\bigr)}_{a_{odjks}}\,
  x_{odjks},
  \label{eq:nominal-obj}
\end{equation}
where $d^{\mathrm{orig}}_{oj}$ is the walking distance from origin $o$ to VBS $j$,
$d^{\mathrm{dest}}_{dk}$ is the walking distance from VBS $k$ to destination $d$,
$c_{jk}$ is the in-vehicle travel time between VBS $j$ and $k$, and $\lambda$
is a weight that converts routing time to the same unit as walking distance.

\subsection{Constraints}

\begin{align}
  \sum_{j,k} x_{odjks} &= 1 & &\forall (o,d), s \label{eq:assign} \\
  x_{odjks} &\le z_{js}, \quad x_{odjks} \le z_{ks} & &\forall (o,d), j, k, s
  \label{eq:active} \\
  z_{js} &\le y_j & &\forall j, s \label{eq:build} \\
  \sum_j y_j &= l, \quad \sum_j z_{js} = k & &\forall s \label{eq:budget}
\end{align}

% ──────────────────────────────────────────────────────────────────────────────
\section{Uncertainty Set for Demand}
\label{sec:uncertainty}

\subsection{Parameterization}

For each OD pair $(o,d)$ and scenario type $s$, let $\overline{q}_{ods}$
denote an upper bound on demand, and let $\overline{Q}_s$ denote an upper bound
on \emph{total} demand in scenario type $s$.  Lower bounds are fixed at zero
(see \S\ref{sec:empirical-uset} for justification).  The uncertainty set is
\begin{equation}
  \cU_s = \Bigl\{
    q_s :
    0 \le q_{ods} \le \overline{q}_{ods}\ \forall (o,d),\;
    \sum_{(o,d)\in\Omega} q_{ods} \le \overline{Q}_s
  \Bigr\}.
  \label{eq:uset}
\end{equation}

This uncertainty set says two things simultaneously:
\begin{itemize}
  \item each OD pair must remain within its own historically calibrated range;
  \item the aggregate demand in the scenario cannot exceed $\overline{Q}_s$.
\end{itemize}

\subsection{Interpretation of the total-demand cap}

\begin{itemize}
  \item If $\overline{Q}_s = 0$, then all OD demands are forced to zero (vacuous).
  \item If $\overline{Q}_s \ge \sum_{(o,d)} \overline{q}_{ods}$, then the total
        demand cap is inactive and the model reduces to independent box bounds
        on each OD pair.
  \item Intermediate values of $\overline{Q}_s$ allow some OD pairs to be high
        simultaneously, but only so long as total demand stays below the
        calibrated cap.
\end{itemize}

A larger $\overline{Q}_s$ for a high-demand period such as evening encodes the
prior that those periods experience larger systemwide demand surges.

\subsection{Remark on integer demand}

Observed demand is integer-valued, but the robust inner maximization may still
be solved as a linear program.  The key point is that the uncertainty set
\eqref{eq:uset} is a bounded knapsack-type polytope.  If the bounds
$\overline{q}_{ods}$ and $\overline{Q}_s$ are integral, the inner LP has an
optimal extreme point that is integral as well.  Hence dropping integrality in
the inner maximization does not change the worst-case value, even though the
uncertainty set is written in continuous form.

\section{Robust Counterpart}
\label{sec:robust-cpt}

\subsection{Robust objective}

The robust model replaces the nominal objective with
\begin{equation}
  \min_{y,z}
  \sum_{s \in S}
  \max_{q_s \in \cU_s}
  \Phi_s(z_s,q_s),
  \label{eq:robust-obj}
\end{equation}
subject to the first-stage constraints \eqref{eq:build}--\eqref{eq:budget},
where $\Phi_s(z_s, q_s)$ is the recourse value defined below.

\subsection{Recourse value function}

The key modeling point is that $x$ is a \emph{recourse} decision.  For fixed
scenario-active stations $z_s$ and realized demand vector $q_s$, define the
recourse value function
\begin{equation}
  \Phi_s(z_s,q_s)
  :=
  \min_{x}
  \sum_{(o,d)\in\Omega}\sum_{j,k\in J}
  q_{ods}\,a_{odjks}\,x_{odjks}
  \label{eq:recourse-value}
\end{equation}
subject to
\begin{align}
  \sum_{j,k} x_{odjks} &= 1 && \forall (o,d), \label{eq:recourse-assign} \\
  x_{odjks} &\le z_{js},\quad x_{odjks} \le z_{ks}
  && \forall (o,d),j,k, \label{eq:recourse-active}
\end{align}
with $x_{odjks} \ge 0$.  Integrality is not needed here: for fixed $(o,d,s)$
the feasible region is a simplex intersected with bound constraints, so an
optimal extreme point selects a single active station pair.

For fixed $z_s$, this problem decomposes by OD pair.  Define the induced
minimum assignment cost
\begin{equation}
  m_{ods}(z_s)
  :=
  \min_{x_{od\cdot\cdot s}}
  \left\{
    \sum_{j,k} a_{odjks}\,x_{odjks}
    :
    \sum_{j,k} x_{odjks}=1,\;
    x_{odjks}\le z_{js},\;
    x_{odjks}\le z_{ks}
  \right\}.
  \label{eq:induced-cost}
\end{equation}
Because all demand coefficients are nonnegative and the recourse constraints are
separable across OD pairs, the recourse value can be written as
\begin{equation}
  \Phi_s(z_s,q_s)
  =
  \sum_{(o,d)\in\Omega} q_{ods}\,m_{ods}(z_s).
  \label{eq:recourse-separable}
\end{equation}
For fixed $z_s$, the inner worst-case demand problem in \eqref{eq:robust-obj}
becomes
\begin{equation}
  \max_{q_s \in \cU_s}
  \sum_{(o,d)} q_{ods}\,m_{ods}(z_s)
  \quad
  \text{s.t.}\;
  0 \le q_{ods} \le \overline{q}_{ods},\;
  \sum_{(o,d)} q_{ods} \le \overline{Q}_s.
  \label{eq:inner-max}
\end{equation}
This is a continuous knapsack problem, most conveniently handled by duality.

\subsection{Derivation of the robust counterpart}

Since $\underline{q}_{ods} = 0$ by design, $B_s := \overline{Q}_s$ exactly,
and the inner maximization \eqref{eq:inner-max} simplifies to a continuous
knapsack:
\begin{equation}
  \max_{r \ge 0}
  \sum_{(o,d)}
  m_{ods}(z_s)\, r_{ods}
  \quad
  \text{s.t.}\quad
  r_{ods} \le \overline{q}_{ods} \;\forall (o,d), \quad
  \sum_{(o,d)} r_{ods} \le \overline{Q}_s.
  \label{eq:inner-max-r}
\end{equation}

\paragraph{Why $m_{ods}(z_s)$ can be treated as a constant.}
The duality is applied only to the \emph{inner} LP, for a \textbf{fixed}
$z_s$.  With $z_s$ held fixed, the set of active stations is known, and
$m_{ods}(z_s)$ \eqref{eq:induced-cost} is a deterministic scalar --- the
minimum feasible assignment cost for OD pair $(o,d)$ given those stations.
The inner LP then has $q$ as its only variable and $m_{ods}(z_s)$ as an
ordinary constant coefficient.  LP strong duality applies to this fixed-$z_s$
inner LP, producing dual constraints that contain $m_{ods}(z_s)$ as a
parameter.  It is only when we re-introduce the outer minimization over $(y,z)$
that $m_{ods}(z_s)$ becomes a nonlinear function of a decision variable; the
$x$-variable substitution in Step 4 handles exactly this.

\paragraph{Step 1: Form the Lagrangian.}
Introduce dual variable $\alpha_s \ge 0$ for the total-demand budget
constraint and $\beta_{ods} \ge 0$ for each per-OD upper-bound constraint.
The Lagrangian is:
\begin{align}
  L(r,\alpha_s,\beta)
  &= \sum_{(o,d)} m_{ods}\, r_{ods}
     + \alpha_s \!\left(\overline{Q}_s - \sum_{(o,d)} r_{ods}\right)
     + \sum_{(o,d)} \beta_{ods}\!\left(\overline{q}_{ods} - r_{ods}\right)
  \nonumber\\
  &= \alpha_s\,\overline{Q}_s + \sum_{(o,d)} \beta_{ods}\,\overline{q}_{ods}
     + \sum_{(o,d)} r_{ods}\,\bigl(m_{ods} - \alpha_s - \beta_{ods}\bigr).
  \label{eq:lagrangian}
\end{align}

\paragraph{Step 2: Take the supremum over $r \ge 0$.}
For fixed $(\alpha_s, \beta)$, the supremum of $L$ over $r \ge 0$ is:
\[
  \sup_{r \ge 0} L
  \;=\;
  \begin{cases}
    \alpha_s\,\overline{Q}_s + \displaystyle\sum_{(o,d)} \beta_{ods}\,\overline{q}_{ods}
    & \text{if } \alpha_s + \beta_{ods} \ge m_{ods} \;\forall (o,d), \\
    +\infty & \text{otherwise.}
  \end{cases}
\]
When the coefficient $(m_{ods} - \alpha_s - \beta_{ods}) > 0$ for any pair,
setting $r_{ods} \to \infty$ drives $L \to +\infty$, so the dual must enforce
$\alpha_s + \beta_{ods} \ge m_{ods}(z_s)$ for all $(o,d)$.

\paragraph{Step 3: Dual LP.}
The dual of the inner maximization is therefore:
\begin{equation}
  \min_{\alpha_s \ge 0,\;\beta_{ods} \ge 0}
  \quad
  \overline{Q}_s\,\alpha_s + \sum_{(o,d)} \overline{q}_{ods}\,\beta_{ods}
  \qquad
  \text{s.t.}\quad
  \alpha_s + \beta_{ods} \ge m_{ods}(z_s)
  \quad \forall (o,d).
  \label{eq:dual}
\end{equation}
Strong duality holds since the primal \eqref{eq:inner-max-r} is feasible
(e.g.\ $r=0$) and bounded.

\paragraph{Step 4: Embed the recourse minimization.}
The dual constraint \eqref{eq:dual} still contains the recourse value
$m_{ods}(z_s)$ from \eqref{eq:induced-cost}, which is itself a minimization
over assignment variables $x$.  We substitute directly: introduce continuous
assignment witnesses $x_{odjks} \ge 0$ satisfying
\eqref{eq:recourse-assign}--\eqref{eq:recourse-active}, and replace
$m_{ods}(z_s)$ with $\sum_{j,k} a_{odjks}\,x_{odjks}$.  The constraint becomes:
\begin{equation}
  \alpha_s + \beta_{ods}
  \;\ge\;
  \sum_{j,k} a_{odjks}\,x_{odjks}
  \qquad \forall (o,d),\, s.
  \label{eq:robust-con}
\end{equation}
Note that no auxiliary $t_{ods}$ variable is needed: \eqref{eq:robust-con}
directly links the dual variables to the assignment cost of $x$.

For any feasible $x$, the right-hand side $\sum_{jk}a_{odjks}x_{odjks}$
is an upper bound on $m_{ods}(z_s)$, so \eqref{eq:robust-con} is
at least as tight as the original dual constraint.  At optimality the
minimization of $\overline{Q}_s\alpha_s + \sum\overline{q}_{ods}\beta_{ods}$
drives each $x_{od\cdot\cdot s}$ to minimize its assignment cost --- a smaller
right-hand side relaxes \eqref{eq:robust-con}, allowing smaller
$(\alpha_s, \beta_{ods})$, which reduces the objective.  Hence the optimal $x$
achieves $\sum_{jk}a_{odjks}x_{odjks} = m_{ods}(z_s)$, and the two
formulations are equivalent.

\paragraph{Robust counterpart MILP.}
Combining with the outer minimization over $(y,z)$, the full formulation is:
\begin{equation}
  \min_{x,y,z,\alpha,\beta}
  \sum_s \overline{Q}_s\,\alpha_s
  + \sum_s\sum_{(o,d)} \overline{q}_{ods}\,\beta_{ods}
  \label{eq:robust-counterpart}
\end{equation}
subject to
\begin{align}
  \alpha_s + \beta_{ods}
  &\ge \sum_{j,k} a_{odjks}\,x_{odjks}
  && \forall (o,d),\, s,
  \label{eq:robust-con-milp}
  \\
  \sum_{j,k} x_{odjks} &= 1
  && \forall (o,d),\, s,
  \label{eq:recourse-counterpart-assign}
  \\
  x_{odjks} &\le z_{js},\quad x_{odjks} \le z_{ks}
  && \forall (o,d),\, j,k,\, s,
  \label{eq:recourse-counterpart-active}
\end{align}
plus \eqref{eq:build}--\eqref{eq:budget}, with $x_{odjks},\alpha_s,\beta_{ods}\ge 0$.

\section{JuMP Implementation Sketch}
\label{sec:jump}

The data required for the robust model (in addition to the base model data) are:
\begin{verbatim}
q_high[od, s]      # per-OD upper demand bound  (q_bar_ods)
Q_cap[s]           # total-demand cap            (Q_bar_s)
a[od, j, k, s]     # assignment cost
\end{verbatim}

New variables ($x$ is \emph{continuous}---a recourse witness, not a demand count):
\begin{verbatim}
@variable(model, x[od in OD, j in J, k in J, s in S] >= 0)
@variable(model, alpha[s in S] >= 0)
@variable(model, beta[od in OD, s in S] >= 0)
\end{verbatim}

Robust objective:
\begin{verbatim}
@objective(model, Min,
    sum(Q_cap[s] * alpha[s] for s in S)
    + sum(q_high[od,s] * beta[od,s] for od in OD, s in S)
)
\end{verbatim}

Assignment witness constraints ($\sum_{j,k} x = 1$, $x \le z$):
\begin{verbatim}
@constraint(model, [od in OD, s in S],
    sum(x[od,j,k,s] for j in J, k in J) == 1)
@constraint(model, [od in OD, j in J, k in J, s in S],
    x[od,j,k,s] <= z[j,s])
@constraint(model, [od in OD, j in J, k in J, s in S],
    x[od,j,k,s] <= z[k,s])
\end{verbatim}

Robust dual constraint (no auxiliary $t$ needed---assignment cost appears directly):
\begin{verbatim}
@constraint(model, [od in OD, s in S],
    alpha[s] + beta[od,s] >=
        sum(a[od,j,k,s] * x[od,j,k,s] for j in J, k in J)
)
\end{verbatim}

The first-stage constraints \eqref{eq:build}--\eqref{eq:budget} are unchanged.

\section{Experimental Setup}
\label{sec:experiment}

We evaluate the model on the Zhuzhou microtransit dataset, which contains
89{,}669 trips recorded over 111--114 days per time-of-day period.  Trips are
partitioned into four scenario types by start hour: morning (06--10), afternoon
(10--15), evening (15--20), and night (20--24).  The calibration script
\texttt{scripts/generate\_demand\_bounds.jl} reads the historical order data,
groups it by period, computes per-OD quantiles and a total-demand cap, and writes
the bounds to \texttt{Data/demand\_bounds.json}.

\subsection{Calibrating the uncertainty set}
\label{sec:calibration}

Group historical observations by scenario type $s \in \{\text{morning},
\text{afternoon}, \text{evening}, \text{night}\}$.  For each OD pair $(o,d)$,
set the lower bound to zero and compute the upper bound as an empirical quantile:
\begin{equation}
  \underline{q}_{ods} = 0,
  \qquad
  \overline{q}_{ods} = Q_{q_{\mathrm{high}}}\!\bigl(q_{od}^{\mathrm{hist}} \mid s\bigr).
  \label{eq:quantiles}
\end{equation}
The rationale for fixing $\underline{q}_{ods}=0$ is discussed in
\S\ref{sec:empirical-uset}.

Next compute the realized total demand on each historical day:
\begin{equation}
  Q^{\mathrm{hist}}_{s,t}
  := \sum_{(o,d)\in\Omega} q^{\mathrm{hist}}_{ods,t},
  \label{eq:hist-total}
\end{equation}
where $t$ indexes historical observations within scenario type $s$.  Then set
\begin{equation}
  \overline{Q}_s
  =
  Q_{Q_{\mathrm{cap}}}\!\bigl(Q^{\mathrm{hist}}_{s,t}\bigr),
  \label{eq:qcap-cal}
\end{equation}
where $Q_{\mathrm{cap}}$ is the chosen calibration percentile (e.g., 0.90).

One practical check is to verify that
\[
  \sum_{(o,d)} \underline{q}_{ods} \le \overline{Q}_s
  \le \sum_{(o,d)} \overline{q}_{ods};
\]
the left inequality holds trivially since $\underline{q}_{ods}=0$, and the
right inequality requires that the chosen $\overline{Q}_s$ not exceed the
sum of all per-OD upper bounds.  When it does, the cap is non-binding and the
robust model degenerates to a pure box model.

\subsection{Empirical analysis of the uncertainty set}
\label{sec:empirical-uset}

\textbf{Lower bounds: $\underline{q}_{ods} = 0$.}
We fix the lower bound at zero for all OD pairs, for two reasons.
First, demand is highly sparse: most OD pairs appear on fewer than 10\% of
days, so a quantile-based lower bound would be zero for the vast majority of
pairs regardless.
Second, and more importantly, $\underline{q} = 0$ is the \emph{conservative}
choice for the operator.  In the robust formulation, $\underline{q}_{ods}$
acts as a floor that \emph{constrains the adversary from below} --- it forces
them to place at least $\underline{q}_{ods}$ demand on every OD pair, even
cheap ones, consuming budget that could otherwise be concentrated on the most
costly corridors.  Setting $\underline{q} = 0$ removes this floor and allows
the adversary maximum flexibility, giving the operator stronger protection.
The practical consequence is that $B_s = \overline{Q}_s$ exactly, and the
$\sum\underline{q}\cdot t$ term in the robust objective vanishes entirely.

\textbf{Upper bounds: $\overline{q}_{ods} = Q_{0.90}$ of zero-padded historical demand.}
The total-demand cap $\overline{Q}_s$ is the 90th percentile of the daily
aggregate demand in period $s$.

\paragraph{Is the total-demand cap binding?}
The central question is whether
$\sum_{(o,d)}\overline{q}_{ods} > \overline{Q}_s$.
If not, the cap is slack even when every OD pair hits its maximum simultaneously,
and the robust model degenerates to a pure box model.  Formally:
\begin{itemize}
  \item \textbf{Non-binding (pathological):}\;
        $\sum\overline{q}_{ods} \le \overline{Q}_s$ — the budget is never
        exhausted; robustness is entirely determined by per-OD bounds.
  \item \textbf{Binding (meaningful):}\;
        $\sum\overline{q}_{ods} > \overline{Q}_s$ — the adversary cannot push
        all OD pairs to their upper bounds at once.
\end{itemize}
Table~\ref{tab:uset-numbers} reports the key quantities; all four periods are
in the binding regime.

\paragraph{What does the adversary do?}
The cap forces the adversary to trade off demand across OD pairs.  To see this,
rank active OD pairs by $\overline{q}_{ods}$ descending and compute the
cumulative sum.  Once this cumulative sum reaches $\overline{Q}_s$, the budget
is exhausted and the adversary must leave remaining OD pairs at their lower
bounds.  Figure~\ref{fig:adv-budget} shows this greedy-filling picture for all
four periods.

historical day alongside the cap $\overline{Q}_s$ and the box ceiling
$\sum\overline{q}$.  Because $\overline{Q}_s = Q_{0.90}(\text{historical total})$,
approximately 10\% of days fall above the cap line (measured: 9.9--10.7\%).
Historical demand never reaches $\sum\overline{q}$ --- this confirms that
the box maximum is a theoretical extreme that does not occur in practice,
and the cap provides a tighter, data-grounded bound.

  cap $\overline{Q}$ (red dashed) and the box ceiling $\sum\overline{q}$

\paragraph{What the model hedges against.}
Figure~\ref{fig:wc-vs-typical} compares, for the top-30 OD pairs by
$\overline{q}$, the adversary's worst-case greedy allocation against the
median historical demand per pair.  On a typical day most pairs are well
below their upper bounds; the adversary concentrates budget into the
highest-$\overline{q}$ pairs.  This illustrates that the robust model
is not hedging against every OD pair spiking at once (impossible under the
cap) but against a targeted surge in the most consequential corridors.

\paragraph{The cap gives a reasonable, data-grounded uncertainty set.}
The key role of the cap is to ensure the uncertainty set is neither trivially
small nor unrealistically large.  Without the cap, the adversary could push
every OD pair to its individual upper bound simultaneously, yielding a total
demand of $\sum\overline{q}$ --- a level that is never observed historically
(Fig.~\ref{fig:demand-vs-cap}, black dotted line).  Setting
$\overline{Q}_s = Q_{0.90}(\text{total demand})$ constrains the adversary to
demand levels that are consistent with what actually occurs on high-demand days,
making the uncertainty set \emph{empirically plausible} rather than a purely
theoretical worst case.

The calibration percentile governs how conservatively this cap is placed:

\begin{center}
\begin{tabular}{lrrrr}
\toprule
Period & $Q_{0.90}$ & $Q_{0.95}$ & $Q_{0.99}$ & $\sum\overline{q}$ (box max) \\
\midrule
Morning   & 110  & 149  & 291  & 173 \\
Afternoon & 377  & 407  & 543  & 504 \\
Evening   & 696  & 870  & 1055 & 936 \\
Night     & 209  & 235  & 403  & 256 \\
\bottomrule
\end{tabular}
\end{center}

All three quantile choices sit strictly below $\sum\overline{q}$ (except morning
where $Q_{0.99} > \sum\overline{q}$, meaning a 99th-percentile cap would be
non-binding).  The default $Q_{0.90}$ is a reasonable starting point: it
corresponds to a high but historically realistic demand level, and the gap
$\sum\overline{q} - \overline{Q}_s$ quantifies how much the cap tightens the
box.  The experiment sweep in \S\ref{sec:cap-sweep} varies this choice to
measure the resulting price of robustness.

\paragraph{Structural observations.}
\begin{enumerate}
  \item \textbf{Lower bounds are zero by design}: $B_s = \overline{Q}_s$ exactly,
        and the $\sum\underline{q}\cdot t$ term in the robust objective vanishes,
        leaving $\overline{Q}_s\,\alpha_s + \sum\overline{q}_{ods}\,\beta_{ods}$.

  \item \textbf{Only 11--18\% of OD pairs are active}: those present on
        $\ge$10\% of days receive $\overline{q}_{ods} > 0$; the rest are excluded
        and assigned zero demand in the worst case.  Rare high-demand spikes on
        sporadic pairs are not hedged.

  \item \textbf{The cap is genuinely binding}: ratios 1.23--1.57$\times$
        confirm the set is not pathological.
\end{enumerate}

\subsection{Two model variants}
\label{sec:variants}

We compare:
\begin{enumerate}
  \item \textbf{Nominal model.}  Uses $q_{ods} = \E[q_{ods}]$ (scenario-average
        historical demand).
  \item \textbf{Robust total-demand-cap model.}  Uses uncertainty set $\cU_s$
        with $\overline{Q}_s$ calibrated from data via \eqref{eq:qcap-cal}.
\end{enumerate}

Each model produces a station selection $(y^\star, z^\star)$.  Given those
station decisions and a realized demand vector, assignment is handled through
the recourse problem \eqref{eq:recourse-value}--\eqref{eq:recourse-active}.

\subsection{Sensitivity to calibration quantiles}
\label{sec:cap-sweep}

The uncertainty set has two independent quantile parameters:
\begin{itemize}
  \item $q_{\mathrm{high}}$: the quantile used for per-OD upper bounds
        $\overline{q}_{ods} = Q_{q_{\mathrm{high}}}(\text{historical OD demand})$.
        Controls how wide the individual OD ranges are.
  \item $Q_{\mathrm{cap}}$: the quantile used for the total-demand cap
        $\overline{Q}_s = Q_{Q_{\mathrm{cap}}}(\text{historical total demand})$.
        Controls how tightly the total demand budget is set.
\end{itemize}

Both are currently set to 0.90 by default (as used in the empirical analysis of
\S\ref{sec:empirical-uset}).  We sweep both independently to understand how
each dimension of conservatism affects the solution:

\[
  q_{\mathrm{high}} \;\in\; \{0.90,\; 0.95\},
  \qquad
  Q_{\mathrm{cap}}  \;\in\; \{0.90,\; 0.95,\; 0.99\}.
\]

\begin{center}
\begin{tabular}{llrrrr}
\toprule
\multicolumn{2}{l}{Cap / OD bound quantile} & \multicolumn{4}{c}{$\overline{Q}_s$ (total-demand cap)} \\
\cmidrule(lr){3-6}
$Q_{\mathrm{cap}}$ & $q_{\mathrm{high}}$ & Morning & Afternoon & Evening & Night \\
\midrule
0.90 & 0.90 & 110 & 377 & 696 & 209 \\
0.95 & 0.90 & 149 & 407 & 870 & 235 \\
0.99 & 0.90 & 291 & 543 & 1055 & 403 \\
\midrule
\multicolumn{2}{l}{$\sum\overline{q}$ at $q_{\mathrm{high}}=0.90$} & 173 & 504 & 936 & 256 \\
\multicolumn{2}{l}{$\sum\overline{q}$ at $q_{\mathrm{high}}=0.95$} & — & — & — & — \\
\bottomrule
\end{tabular}
\end{center}

The expected effects:
\begin{itemize}
  \item \textbf{Raising $Q_{\mathrm{cap}}$}: larger $\overline{Q}_s$ $\to$ adversary
        has more total-demand budget $\to$ worst-case cost rises $\to$ more
        conservative design, higher in-sample cost, stronger out-of-sample
        tail guarantee.
  \item \textbf{Raising $q_{\mathrm{high}}$}: wider per-OD intervals $\to$ $\sum\overline{q}$
        increases $\to$ the gap $\sum\overline{q} - \overline{Q}_s$ grows, so the
        total-demand cap does \emph{more} work relative to the box.  Effectively,
        the adversary has a larger individual budget per corridor.
  \item \textbf{Joint effect}: raising both simultaneously yields the most
        conservative model; the experiment isolates each dimension.
\end{itemize}

Combined with three station budgets $k \in \{6, 8, 10\}$ and two model variants,
the full sweep is $2 \times 3 \times 2 \times 3 = 36$ jobs.

\subsection{Out-of-sample evaluation protocol}
\label{sec:oos}

For each held-out demand realization $q^{\mathrm{test}}_{ods}$ (from a
held-out date range or a parametric simulation), compute the realized recourse
cost:
\begin{equation}
  \mathrm{Cost}(y^\star,z^\star,q^{\mathrm{test}})
  =
  \sum_{s \in S}
  \Phi_s(z_s^\star, q_s^{\mathrm{test}}).
  \label{eq:realized-cost}
\end{equation}
Equivalently, one may evaluate
\[
  \sum_{s,(o,d)} q_{ods}^{\mathrm{test}}\,t^\star_{ods},
\]
where $t^\star_{ods}$ denotes the minimum assignment cost induced by the fixed
active stations in scenario $s$.

\subsection{Reported metrics}

For each model, report over the test set:
\begin{itemize}
  \item Mean realized cost
  \item 90th and 95th percentile realized cost
  \item Worst-case realized cost
  \item Number of OD pairs poorly assigned (realized walking distance exceeding a threshold)
  \item Stability of station selection across demand realizations
  \item \emph{Price of robustness}:
        $\displaystyle\frac{\text{Robust obj} - \text{Nominal obj}}{\text{Nominal obj}}$
\end{itemize}

\subsection{Expected finding}

The robust total-demand-cap model protects against systemwide demand surges
without assuming that every OD pair simultaneously reaches its upper bound.  It
should improve high-percentile out-of-sample performance relative to the
nominal model, while typically accepting some increase in mean in-sample cost.

Formally:
\[
  \text{Nominal P90} \;\ge\; \text{Robust-Cap P90},
\]
\[
  \text{Nominal mean} \;\le\; \text{Robust-Cap mean}.
\]

\subsection{Experiment pipeline}

The experiment is driven by scripts in \texttt{scripts/}:

\begin{enumerate}
  \item \textbf{\texttt{scripts/generate\_demand\_bounds.jl}}: reads historical
        order data, groups by time-of-day period, calibrates per-OD quantiles
        and total-demand caps, and writes \texttt{Data/demand\_bounds.json}.
  \item \textbf{\texttt{scripts/01\_setup.jl}}: reads
        the experiment base config and sweep config, then generates per-job
        TOML configs.
  \item \textbf{\texttt{scripts/02\_submit.sh}}: SLURM array job template.
  \item \textbf{\texttt{scripts/03\_run\_job.jl}}: loads config, runs the

  \item \textbf{\texttt{scripts/aggregate.jl}}: loads all \texttt{metrics.json}

\end{enumerate}

The sweep in \texttt{experiments/2026-05-01\_nominal\_vs\_robust/} crosses

$k \in \{6, 8, 10\}$, giving $2 \times 3 \times 2 \times 3 = 36$ jobs in total.

\section{Conclusion}
\label{sec:conclusion}

We propose a robust extension of the VBS station selection model based on
per-OD upper bounds together with a scenario-level cap on total demand, with
lower bounds fixed at zero by design.  The key insight is that, once assignment
is written correctly as a recourse function, the resulting worst-case demand
problem is a continuous knapsack in demand variables, with a clean dual that
yields a linear robust counterpart solvable by the same MILP solver as the
nominal model.  The parameters $\overline{q}_{ods}$ and $\overline{Q}_s$ have
direct empirical interpretations and can be calibrated transparently from
historical data.  Framed this way, the report isolates the core comparison of
interest: the nominal base model versus a tractable robust counterpart that
hedges against adverse but bounded demand shifts.
